# 无人机日志管理系统 - 需求分析与实施方案

## 一、需求理解与业务背景

### 项目定位
本系统是为公司内部管理设计的无人机日志管理系统，核心目标是：
1. **工作量记录** - 追踪每位员工的工作产出
2. **报表导出** - 生成月报、日报供管理层审阅
3. **绩效考核** - 基于巡查数据评估员工绩效

### 与客户需求的差异
客户提到的是"局上"的巡查系统（政府监管部门），但公司内部实际需要的是一个**内部管理工具**，重点在于：
- ✅ 员工工作量统计
- ✅ 巡查报告管理
- ✅ 报表导出（月报、日报）
- ✅ 绩效数据分析
- ❌ 非面向公众/政府的外网系统

## 三、技术架构评估

### 当前技术栈
- **前端**：React 18 + TypeScript + Vite
- **UI**：纯CSS自定义（无UI框架）
- **后端**：Supabase（PostgreSQL + Auth + Storage）
- **状态管理**：React Context
- **路由**：React Router

### 技术债务与风险
1. **无UI组件库** - 开发新页面需编写大量CSS，周期长
2. **纯前端图片水印** - 安全性不足，建议后端处理
3. **Word导出** - 需要引入新库（docx.js），增加包体积
4. **权限校验** - 目前仅前端控制，需补充RLS策略

### 推荐技术方案

| 功能 | 推荐方案 | 备选 |
|------|----------|------|
| Word导出 | docx.js + file-saver | 服务端生成 |
| 图片水印 | canvas前端绘制（简单） | Supabase Edge Function |
| 图表 | Recharts（已在用） | Chart.js |
| 日期处理 | date-fns | moment.js |
| 性能优化 | React.memo/useMemo | 虚拟列表 |

## 四、数据模型扩展

### 新增表建议

```sql
-- 巡查报告增强（logs表扩展）
-- 使用JSONB的issues字段扩展现有结构

-- 报告审核记录
CREATE TABLE report_reviews (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  report_id UUID REFERENCES logs(id),
  reviewer_id UUID REFERENCES auth.users(id),
  action VARCHAR(20), -- 'approve', 'reject', 'return'
  comment TEXT,
  created_at TIMESTAMP DEFAULT NOW()
);

-- 年度巡查计划
CREATE TABLE annual_plans (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  year INTEGER,
  base_id UUID,
  target_inspections INTEGER,
  target_area DECIMAL,
  created_by UUID REFERENCES auth.users(id)
);

-- 任务转发记录
CREATE TABLE task_forwards (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  report_id UUID REFERENCES logs(id),
  forward_to VARCHAR(100), -- 责任单位
  template TEXT,
  photos TEXT[],
  status VARCHAR(20), -- 'pending', 'received', 'resolved'
  created_at TIMESTAMP DEFAULT NOW()
);

-- 绩效记录
CREATE TABLE performance_records (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES auth.users(id),
  month VARCHAR(7), -- YYYY-MM
  inspection_count INTEGER,
  issue_count INTEGER,
  report_quality_score DECIMAL,
  total_score DECIMAL,
  calculated_at TIMESTAMP DEFAULT NOW()
);
```

## 六、关键问题与建议

### 问题1：Word导出格式
**建议**：与客户确认Word模板格式，特别是：
- 公司抬头/页眉页脚
- 报告编号格式
- 表格样式
- 签章位置

### 问题2：绩效计算规则
**建议**：明确绩效算法，例如：
- 巡查一次得多少分？
- 发现问题如何加权？
- 是否有扣分项？

### 问题3：任务流转范围
**建议**：确认"任务转发"的使用场景：
- 转发给谁？（内部部门/外部单位）
- 对方是否使用本系统？
- 是否需要确认/反馈机制？

### 问题4：数据来源
**建议**：确认巡查数据的来源：
- 无人机飞行数据自动导入？
- 还是完全手工录入？
- 是否对接无人机厂商API？
